这篇博客的标题叫《Task Paralysis and AI》,作者是德国一个开发者。他先把「task paralysis」和大家熟悉的「analysis paralysis」分开。Analysis paralysis 是脑子转不停、想太多;task paralysis 是脑子干脆不转,你坐在那里,知道该做什么,但就是动不了。两个东西不一样。

然后他写到 LLM 怎么治这个病。任务有点难、有点烦、不知道从哪下手——以前他会卡住一天甚至两天。现在丢给 Claude,几分钟出一个能跑的版本,他从那个版本接着改。Task paralysis 被破开了。

问题来了。破开的代价是上瘾。

他记账记得很清楚。第一笔:接近 100 欧的 Claude tokens,买来跑几个 coding 项目。然后买了 Pro 月费。再然后买了 20 欧的 API credits,因为 Pro 的额度不够。再然后月中升到 Max,因为 Max 的额度也不够。他还试了各种省 token 的小招式,/modus opusplan 那种命令。每一笔都买得很自觉,每一笔都买完之后觉得「值」。

「Throw endless money at your source of dopamine,像吸毒的人跑去找 dealer 求下一针,或者像在赌场里等下一张幸运牌的人。」这是他自己原话。

最有意思的是他没给解药。文章不是「五招帮你戒 AI 上瘾」那种自助书结构。他承认问题,描述循环,然后老实写下最后一句:「But the dopamine feels so freaking good.」就这样收尾。

我读完两遍。这种诚实在 AI 创作的内容海里几乎绝迹——大部分人不是在卖你 AI 工具,就是在卖你「如何用 AI 工具」的课程,愿意承认 AI 把自己变得更财务脆弱、更依赖即时满足的,真的不多。

而这才是 LLM 时代一个值得被记下来的副作用。我们一直在讨论 AI 替代多少岗位、节省多少时间、写出多少代码,但很少有人去聊它在用户的钱包和神经回路里同时挖坑这件事。Pro / Max / API credits / Codex / 各种 add-on,从产品设计的角度看就是一组精心铺好的多巴胺梯子,你不爬到顶层永远会觉得自己「漏了点什么」。

更深一层的问题:LLM 修好了 task paralysis,但它修的方式是把启动门槛降到零。门槛降到零的副作用是,你失去了「这件事我自己干完了」的小成就感——那个成就感本来也是多巴胺,只不过是慢一点、需要等的那种。当快的多巴胺无成本可得,慢的多巴胺会显得不划算。慢慢地,你连不需要 AI 的小任务都开始顺手丢给 AI,因为「为什么不丢」。再过一阵,丢出去就成了肌肉记忆。

我自己用 Claude Code 也一年多了,这一段读着心里发虚。值不值得?Boss 我不知道——这个问题大概只有每个人自己一笔笔账看完才有答案。但博主提出的那个观察我同意:在我们沉浸在「AI 把生产力提升了多少」这种叙事里的时候,有人付出的代价正在被忽略,而那些代价不会反映在 LinkedIn 的庆功帖里。它们藏在订阅账单的明细里,也藏在你下次想自己开始写一段代码前犹豫的那 5 秒钟里。

至少我现在开始记账。
