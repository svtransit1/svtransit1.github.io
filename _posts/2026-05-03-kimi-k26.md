先说几个数字。Kimi K2.6 这一版的官方披露里,最显眼的是它能连续做 13 小时的编码任务,过去的大模型在中等长度任务里就开始上下文断裂、逻辑前后矛盾,这一档直接拉到大半个工作日。它的 agent 集群可以同时调度 300 个 sub-agent 并行,跨 4000 个协作步骤——意思是你给它一个项目级目标,它能自己拆任务、分发到不同 sub-agent、收回结果再交付。这不是 chat-bot 形态,是一个能跑半天的执行体。

往参考系一看就更明白这件事的位置。Anthropic 的 Claude Code 现在能在 1250 万行代码的仓库里完成特定任务,大约 7 小时。OpenAI 的 Agent Swarm 架构主打多 agent 协作做复杂研究。智谱(Zhipu·AI)这个月发的开源旗舰也在往长链条任务这边走。Kimi K2.6 这次是把"长任务执行能力"做成核心叙事——三家在做同一件事,但各走各的路:Anthropic 偏向"在大型 codebase 里精准动刀",OpenAI 偏向"开放给开发者搭自己的 agent",月之暗面这次偏向"开箱即用的 300 agent 工作流"。

资本层面值得停下来看的是估值跳跃。月之暗面 2 月那一轮的估值是 100 亿美元,Bloomberg 最近报的预 IPO 估值是 180 亿美元,意味着两个月里又跳了 80%。北京公司这边正在准备港股 IPO,同时还在募集大约 10 亿美元的新一轮。对比一下,智谱在港股已经上市,市值大约 4347 亿港币,折合约 555 亿美元——意味着月之暗面如果按 180 亿美元 IPO,跟智谱之间还差一个数量级,留出的成长空间反而是优势。中国 AI 创业公司里,头部第二梯队跟第一梯队之间能差到 3 倍以上市值的不多见,这也意味着资本市场对"谁能率先证明长任务能力跟商用化路径"还在等结论。

谁该跟一下这条新闻。如果你已经在用 Claude Code 或 Cursor 做工程级任务,Kimi K2.6 是中国这边唯一一个可以跟它们摆在一起对比的开源模型,值得装下来跑一跑你最复杂的那个 repo。如果你做 agent 框架研究,300 sub-agent + 4000 协作步骤这个规模是开源世界里少见的;过去这种集群多见于大公司内部 demo,Moonshot 直接开源出来,意味着外面的研究者可以做更深入的 agent-orchestration 实验。

实操层面有几个细节值得提。Kimi 系列从 K1 到 K2 再到这次 K2.6,过去一年迭代节奏明显加快——从月级别的小幅优化,变成季度级别的能力大跳跃。这跟硅谷模型的迭代节奏(GPT-3 到 GPT-4 之间隔了大半年,GPT-4 到 GPT-5 又隔了一年多)是两条不同的曲线,Moonshot 走的是高频小步迭代,这种打法在国内更适合追赶。开源策略也是个加分项:K2.6 的权重直接放出来,相当于给了第三方框架开发者一个对标 Anthropic / OpenAI 的免费基础设施。

对比维度可以拆得更细一点。在"任务时长"这一档,Kimi K2.6 的 13 小时连续编码 vs Claude Code 的 7 小时大型 codebase 任务,数字看起来 Kimi 更长,但要注意两者的衡量基准不同——Claude Code 是在固定 1250 万行代码上的特定任务时长,Kimi K2.6 是泛指"持续编码"。直接对比意义有限,但都说明同一件事:大模型已经能扛住一个工作日的连续工程操作,这是过去半年里没真正实现的能力。

有一件事值得拆出来单独看。"长任务执行"这一档的能力,过去半年是被反复提的方向,但真正把 13 小时连续编码做出来并且开源,Moonshot 这次是开了个口子。这说明大模型的竞争主线正在从"单次回复质量"变成"多步骤任务的稳定性",未来一年的 benchmark 选择、生产部署评估、价格模型,都会往这个方向重新校准。

如果让我现在做选型决定,英语圈深度集成场景仍然推 Claude Code,代码改动对生产的影响最大,精准度优先;但中文圈、对成本敏感、想自己搭 agent 编排的团队,Kimi K2.6 的开源属性 + 长任务能力让它成为新的可选项。月之暗面这一波拿到 IPO 节奏 + 长任务能力的双护城河,接下来一两年值得长期跟。
