每个工程师都在 Cursor 里跑、PM 都在 Claude 里写需求、Sales 都在 ChatGPT 里改邮件。个人产出明显涨了。但回头看公司层面 —— 流程没改、决策没快、产品形态没变。Glaser 把这叫"messy middle 困境"：AI 使用的颗粒度已经下降到"工作流里的某一段循环"，但传统的组织学习机制（brown bag、champion network、迭代复盘）还停留在团队级别，根本捕捉不到。

最值得记的是他提的那一组指标 —— "token-to-learning" 而不是 "token-to-output"。意思是别问"今天 agent 帮你跑了多少代码"，要问"agent 跑过这一遍之后，公司知道了什么以前不知道的事？"。前者是产出，后者是壁垒。

他列了三个被忽视的能力：第一，Agent Operations，给 agent 加治理边界；第二，Loop Intelligence，识别哪些 agent 循环会带来学习；第三，Agent Capabilities，把有用的能力跨团队复用。中国 AI 圈最近所有讨论都集中在前两步 —— 招 prompt engineer、选 LLM 厂商、配 RAG。第三步几乎没人聊：内部知识管理 + 跨团队能力分发，才是把"个人产出"变成"组织能力"的关键齿轮。

最锐利那一刀在 surveillance 风险那段：如果 agent 系统变成员工打分工具，team 会主动隐藏实验、隐藏失败 —— 学习就停了。这点对国内大厂尤其值得想想，KPI 文化一旦把 agent 使用率纳入考核，组织学习反而会下降。

我自己的判断：下半年 AI 转型的胜负手不在工具选型，在于谁能把"个人产出"和"组织资产"连起来。每个人都用得很爽的公司，往往学得最慢。
