1. 视觉能力跳了一大截

XBOW 的 visual-acuity benchmark 上,Opus 4.6 拿 54.5%,Opus 4.7 拿 98.5%。长边最大支持 2576 像素,大概 3.75 MP,是之前的 3 倍。意思是以前给模型看高清截图要先手动 resize,现在大部分 UI 截图、设计稿、论文图可以直接丢进去。

2. Rakuten-SWE-Bench 真实任务 3 倍

这个 benchmark 是拿 Rakuten 线上的 production 任务跑的——不是合成数据。4.7 比 4.6 解决了 3 倍的任务量。93 道题的通用 coding 榜也有 +13% 的提升,CursorBench 从 58% 跳到 70%。

3. 多了一个 xhigh effort tier

原来只有 high / max,现在中间插了一个 xhigh(extra high),给硬题更精细的推理-延迟权衡。默认情况 Claude Code 现在跑的就是 xhigh——也就是说,订阅 Pro/Max 的你,从升级那天起 agent loop 变深了,每个 task 多烧一点 token,但成功率上去了。

4. /ultrareview 这个 slash 命令

新加的 code review 专用 slash。Pro 和 Max 用户每月送 3 次免费 ultra review。对开一个 PR 就要 review 一次的团队,这一条等于帮你省了 3 次人工。

5. 要注意的坑:tokenizer 换了

这一条很少人注意——Opus 4.7 的 tokenizer 做了更新,内容类型不同,token 映射变成原来的 1.0 到 1.35 倍。意思是同一段文本,新模型计 token 可能多 35%。如果你有按 token 预算烧钱的脚本,发布第一周就该重新校准。很多人发现账单莫名涨了,根因是这里。

价格没动:$5 / 1M 输入 token,$25 / 1M 输出。API 在 AWS Bedrock、GCP Vertex AI、Microsoft Foundry 都上线了,三家云任选。

合起来看,这次发布的重心明显不在 demo 视频那种场景,而是把 agent 路径推深——98.5% 视觉加上 3 倍 Rakuten production 任务量,再加上默认 xhigh,一次性把长链路 task 的成功率抬了一个台阶。如果你主力用 Claude Code 跑长 task,这一周应该已经感受到差别了。
