Andrej Karpathy 最近在 X 上丢了个东西出来：一份叫「idea file」的文档，把他自己的第二大脑工作流完整写出来给别人抄。过去几个月，他靠这套流程跑出了 100 多篇互相链接的文章，总字数 40 万，几乎没怎么手改。

没有 RAG，没有向量数据库，没有 Obsidian 插件。就 markdown + 一个 agent。

01 把原料扔进去，不整理

看到什么顺手存什么——推文、播客截图、会议笔记、闪念——一律丢进一个 raw 文件夹。不分类、不打标签、不写摘要。整理是 agent 的工作。

02 agent 的角色是图书管理员

Karpathy 的 agent 干的是归类活——挖联系、合并相近主题、给出结构建议。写作的决定权在人手上。

03 markdown 是主存，不是 staging

所有成品都是 markdown 文件，互相用 wiki link 串起来。没有数据库。换工具零成本，手动查找零门槛。

04 代码分享过时了，idea 分享是新常态

他的原话：agent 时代共享代码意义不大——agent 自己会写。稀缺的是结构化的想法。所以把「怎么想」这件事的流程文档化。

05 每一步都可以用 LLM 重跑

因为数据全是纯文本、结构全是扁平 markdown，随时可以把整个工作流迁到新模型上重跑。锁定风险等于零。

40 万字是个实打实的数字。不是那种「工具很神」的空说。
