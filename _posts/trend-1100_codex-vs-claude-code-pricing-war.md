Sam Altman 昨晚发推:"codex is the best AI coding product"——接下来 30 天,任何想从别家切过来试 codex 的公司,免费用 2 个月。原话就是冲着 Claude Code 来的。几小时后,Anthropic 那边 @ClaudeDevs 反手发图:Claude Code 周限额提高 50%,持续到 7 月 13 日,Pro / Max / Team / Enterprise 用户已经自动生效。一张超大字体的"50% higher"海报。15K 点赞,130 万阅读。

两边一起打的是同一群人——已经在用 AI coding agent 写代码的 dev。

价格那一刀。Codex 这边是"给企业 2 个月免费试用"——核心客户群是从 Claude Code 迁过来的团队。Claude Code 那边没动价,而是把限额放宽 50%——核心是让已付费用户少撞墙,降低换 codex 的诱因。两边都没打"哪个模型更聪明",都在打"用着不卡 + 不肉疼"。

切换成本。Codex 押对了一个洞:Claude Code 用户最痛的是限额——做大项目跑着跑着就被"今周配额用完"打断。给 2 个月免费,等于给"试一下别家会不会顺"开了个口子。Anthropic 也看到这个洞,所以直接把限额抬上去——堵住了切换的动机。

时间窗口。Sam 的 30 天报名期 + Codex 2 个月免费 = 90 天客户挖角窗口,几乎卡在 Anthropic 7 月 13 日的提额到期那一天前后收尾。Claude Code +50% 持续到 7 月 13 日 = 大约 60 天的"试试看再决定"缓冲。两边的时间表高度重叠,意味着接下来这两个月,任何团队的"我们到底用哪个"会议,都会被这两条消息撑住——不做决定的成本反而比换工具更高。

editorial 角度看,这件事最有意思的地方,不是"谁会赢",是 AI coding 工具市场终于进入"按使用强度收钱"的阶段——以前是 token 计费,现在变成"我给你够用的额度,别为了省钱多打开一个 tab"。这种打法在 SaaS 时代叫 fair-use pricing,现在被搬到 AI 工具上,意味着市场已经过了"卖访问权"的早期阶段,进入"卖使用密度"的中期。

要选哪边。团队主要做大型 codebase、经常撞 Claude Code 限额的——Codex 这 2 个月免费值得试一下,反正成本只是切个 IDE 配置。已经在 Claude Code 流程里跑得顺的——这 60 天的 50% 提额是白送的福利,不动也行,等 7 月再看新一轮报价。说到底,两边都怕你的注意力被对方占走——这种时候,用户是赢家。

转给那个还在拿 Claude Code 跟你抱怨限额的同事。
