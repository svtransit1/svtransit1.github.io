五年开发,Zed 这个 IDE 一直在 0.x 死磕,这次 1.0 终于 production-ready,Mac、Windows、Linux 三平台都齐了。HN 一夜窜到 1,492 分。值得花 5 分钟看清楚为什么这次发布跟之前不一样 — 它不是又一个 Cursor 替代品,是从底层往上重新想"AI-native IDE 应该长什么样"的一次尝试。

第一件事:GPU 渲染 + Rust。整个 UI 跑在 GPU 上,不是 Electron,不是 webview。打开十万行的项目、滚动巨型 diff、跑 600fps 都不卡。对比 VSCode 这种 Electron 底子的编辑器,Zed 在 M1 Max 上的体感差距大概是"feel like native vs feel like web"。代价是早期插件生态薄,但快是真的快。这跟 Sublime、TextMate 那一代纯本地编辑器的回归感很像 — 只是这次 native 性能加上了 AI 一等公民。如果你之前因为 VSCode 慢卡而退回到 vim/emacs,Zed 是值得重新评估的选项。

第二件事:Agent Client Protocol(ACP)。这是 Zed 1.0 真正的杀招。之前每个 AI IDE 都在自己的 client 协议里锁住 agent — Cursor 的 Composer、Claude Code 的 skills、Copilot 的 chat — 互不兼容。Zed 这次发了一个开源 protocol,任何兼容的 agent 都可以即插即用,Claude Agent、Codex、OpenCode、Cursor 全部已经接入。如果这个 protocol 被社区接受,IDE 跟 agent 之间的耦合就被解了。这条线跟 MCP 早期发力的逻辑很像 — Anthropic 用 MCP 把 model 跟 tool 之间的边界开放化,Zed 这次用 ACP 把 IDE 跟 agent 之间的边界开放化。两个开放协议合起来,未来工具栈的可组合度上一个台阶。

第三件事:并行多 agent 支持。一个项目里可以同时跑两个或更多 agent — 一个改 frontend、一个改 backend、一个跑 test、一个 review,互不打架。底层用 git worktree 做隔离,agent 各自在自己的 working tree 里写代码,合并由你拍板。这一点是 Cursor、Claude Code 现在都没真正做到的。意义在于 agent 协作模式从"串行单 agent"走到"并行多 agent",生产力上限被重新定义。原本一个 PR 要拆成多步交给 agent 做,现在可以一次开三个 worktree 让三个 agent 平行跑,完成时间大幅缩短。

第四件事:DeltaDB 实时同步。这是 Zed 自己造的 sync engine,人 + AI 在同一个文件上 real-time 协作 — 你打字、agent 改代码,差异通过 DeltaDB 推到 server,所有 client 同步看到。听起来像 Google Docs 加 Cursor,但底层数据结构是 CRDT-style merge,理论上可以跨多人 + 多 agent 无冲突合并。下一阶段团队跟 agent 共享 state 的方式,可能就长这样。Cursor 的 Composer 也在做类似的事情,但 Zed 的协议是开源的 — 这点很关键。

第五件事:开源 + 不要广告。Zed 团队明确写了核心 IDE 永远开源,商业模式是协作云服务,不靠 IDE 本身赚钱。这条线跟 Cursor 走付费订阅不一样。中文圈独立开发者特别值得注意 — Cursor 的 $20/月在国内信用卡续费很麻烦,Zed 本地完全免费。换句话说,你想用付费 AI 模型(Claude / GPT)就掏 API key 自己付,Zed 本身不抽成、不限速、不锁功能。这种纯净的工具关系,跟过去十年 SaaS 时代的"工具厂商当中间商"完全反着走。

等等再说的:插件生态。VSCode 三万多个插件,Cursor 兼容大部分。Zed 1.0 自己的插件生态还在早期,你常用的 prettier、eslint、git 集成基本都有,但小众工具可能要等一两个月。如果你的工作流极度依赖某个 niche extension,先试 Zed 但保留 VSCode。这是一个"用得越久越值得切"的工具,前两周可能会觉得有点不顺手,但底层架构的优势是要时间来兑现的。

实操建议:这周下载试一下,先把它跟 Cursor / VSCode 并排开,挑一个真实项目跑半天。重点感受 GPU 渲染的丝滑、agent 集成的顺手、并行 agent 的体感。三件事任意一件让你"回不去",就值得切换。如果三件都没打动,Zed 还不到时候,继续观察。这种"五年磨一剑"的产品发布节奏,跟现在 AI tooling 圈子里"每个月一个新 IDE"的喧闹形成强对比。Zed 团队选了一条慢路,1.0 之后这条线值得长期盯。
