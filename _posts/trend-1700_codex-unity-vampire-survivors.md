这是 Threads 上 @0x0funky 这两天讲的一个亲身实验。最戳人的不是"AI 能写代码"——这件事一年前已经定型——是"我没用过 Unity 也能做出一个完整玩法,因为我不需要懂引擎,我只要描述清楚我要什么"。这把"游戏开发的入门门槛"从"得先学引擎、得先学美术、得先学动画"拍成"得先描述清楚 + 让 AI 跑"。

里面值得拆开看的几条。一是按作者描述的结果,Codex + GPT-5.5 跑通了一个完整 Unity 工程,包括 prefab、scene、animator state machine 这些非代码资产的配置——具体颗粒度作者没全部公开,但跑出能玩的成品是事实。二是 agent-sprite-forge 这个工具按作者的说法,把美术生成接进了 Codex 工作流,大幅减少了在 ChatGPT 跟 Unity 之间手动倒腾素材的步骤。三是十四小时里九成是 idle 时间——你扔出来需求,然后去做别的事,过半小时回来看看进度。这种"非全程操作"的工作模式,过去半年是 agent 圈一直在喊但很少有真实 demo 的形态。

如果你做游戏、做独立产品、或者就是想"看看一个非游戏开发者用 AI 能做出什么样的完整东西",这个 Threads 帖子值得花十分钟看一遍。这不是某个工具厂商的演示,是一个普通开发者用现有的工具拼出来的东西——这种亲历感跟厂商 demo 完全两个量级。
