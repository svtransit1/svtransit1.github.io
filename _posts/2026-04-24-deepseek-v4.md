9 天之内,前沿模型三连发:4 月 16 日 Claude Opus 4.7 → 4 月 23 日 GPT-5.5 → 今天 DeepSeek V4。三家公司、三条产品线、一周一代。以下是 V4 真正值得看的几条:

1. **开源旗舰,Apache 2.0**

V4-Pro 是 1.6 万亿总参数的 MoE,每 token 激活 490 亿。V4-Flash 是效率档,2840 亿总参数、130 亿激活。两条线都是 Apache 2.0,可商用、可微调、可自部署——这个协议是今年头部开源模型里最宽松的。

2. **SWE-bench 80.6%,离 Opus 4.6 只差 0.2 分**

在 SWE-bench Verified 这条代码基准上,V4-Pro 80.6%,Claude Opus 4.6 是 80.8%。0.2 分的差距在统计意义上基本等于打平。注意不是 4.7 比较——4.7 的公开对照数字还没完全出齐。

3. **Codeforces 3206,算法竞赛端接近人类顶端**

V4-Pro 的 Codeforces rating 3206,排进人类榜前 23 名。前面几位基本都是靠选拔进了全球奥赛的选手。纯算法推理这一块,开源模型第一次跨进这个档位。

4. **上下文 1M,整套代码库一次喂进去**

1 百万 token 上下文,对标 GPT-5.5 和 Opus 4.7 的 1M 档。配合他们新提的 Hybrid Attention Architecture,在长对话里 retain 查询的能力比上一代明显更好。这一条对做 Claude Code / Codex CLI 这类长程 agent 的人,最有用。

5. **价格才是真正的护城河 · 差了一个数量级**

API 定价:Pro 输入 $1.74/M token,输出 $3.48/M。Flash 输入 $0.14/M,输出 $0.28/M。作对比——Opus 4.7 是 $5 / $25,GPT-5.5 价格档在同一区间。V4-Flash 的输入价格比 Opus 4.7 低了接近 36 倍。做 BTB batch、做低成本 agent 编排、做成本敏感的内容管线,这个差价会直接决定谁能跑得下来。

6. **33 万亿 token 预训练,数据量是第二代关键变量**

V4-Pro 在 33 万亿 token 上预训练,V4-Flash 在 32 万亿。这个量级往后,模型差异主要不在参数规模,而在训练数据的质量和覆盖面——中文语料的权重在这里是一条悄悄的加分线。

容易忽略的一条:**Engram 记忆架构**没有被主流媒体浓墨渲染,但这是 V4 跟前代 V3 最大的分层差异——模型在长对话中有了"长期记忆分层"的处理路径,不只是靠扩大 context window 硬怼。这个方向如果做对,明年头部开源模型可能都会跟进。

综合看下来的判断:V4 没有在任何一条基准上"超越"前沿闭源,但它在差距拉近到误差范围内的同时,把价格打到了 1/10 的量级,还附带 Apache 2.0。对中小团队、对 API 重度用户、对想自部署的人来说,这不是"还行的备选",是"新的默认"。前两天还在纠结 OpenAI vs Anthropic 选哪家的人,这个周末可能要重新列一张选型表了。
