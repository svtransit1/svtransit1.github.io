Hugging Face 最近放出来一个叫 ml-intern 的 agent,直接答这个问题。官方仓库,3100 个 star,默认跑 Claude Opus 4.6。

定位很明确:把「想跑个实验」到「模型训完推回 HF Hub」之间的那堆琐事自动化。不是新 IDE,也不是新 Chat 前端,是一个专门喂熟了 HF 文档和 API 的 agent runtime。你敲一句 ml-intern "fine-tune llama on my dataset",它自己去查 HF 的 doc、读相关 paper、在 repo 里找现成代码、写脚本、用你授权的 HF Jobs 额度跑 fine-tune,完了再写 model card 推上去。

几个工程上的细节值得看。

上下文到 170k token 会自动 compact——长会话跑几十轮 agent loop,context 溢出是最常见的翻车点,HF 把这一条硬编码进了 runtime。

有 doom loop detection——如果 agent 连续几次在同一个工具、同一个参数上打转(典型的「看不到错误所以一直重试」),runtime 会识别并强制改方向。300 iterations 是硬上限,超过直接停,防止烧钱。

敏感操作需要用户 approval——rm、cloud job submit、sandbox 里执行 arbitrary code,这些都会停下来问一句。默认不自动跑危险命令。

技术栈是 MCP server + GitHub code search + 本地 sandbox + HF API,都是能单独替换的模块。你想把 Claude 换成 GPT-5,改个 flag 就行;不想用 HF Jobs,换成自己的 GPU 集群也可以。

跟 Claude Code 的关系——它不是替代品。Claude Code 是通用 coding agent,ml-intern 是专门给 ML 场景训练过的 system prompt + 工具链。对做模型训练的人来说,ml-intern 省的那一层「把 HF 文档翻一遍再动手」的时间,比花钱订 Opus Max 还划算。

HF 官方下场做 agent 这件事本身也是个信号。过去两年 HF 一直在 infra 层(Transformers、Datasets、Spaces、Hub),现在往上探到 agent 层,意味着「模型训练」这个动作正在从「人 + docs」变成「agent + docs」——人的角色从执行者变成审批者。
